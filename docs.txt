ТАЙМ-МЕНЕДЖЕР — ДОКУМЕНТАЦИЯ
==============================

СТРУКТУРА ПРОЕКТА
-----------------
task.py          — модели данных
interface.py     — графический интерфейс (Flet)
expectations.py  — целевые константы (свободное время, удовлетворение и др.)
tasks.json       — хранилище задач (создаётся автоматически)
requirements.txt — зависимости


ЗАПУСК
------
    pip install -r requirements.txt
    python interface.py


МОДЕЛИ ДАННЫХ (task.py)
-----------------------
TASK_TYPES: list[str]
    Допустимые типы задач: ["учёба", "дела", "хобби"].

class Task (абстрактный, dataclass)
    deadline: Optional[date]    — дедлайн задачи
    time_required: int          — оценочное время выполнения в часах
    type: str                   — тип из TASK_TYPES
    min_cont_interval: int      — минимальный непрерывный рабочий интервал в часах
    is_done: bool               — закрыта ли задача (всегда False при создании)

class Work(Task)
    Задачи-дела. Метод Complete() помечает задачу выполненной.

class Learning(Work)
    Задачи-учёба. Метод MatchWith(other) — совместимость с другой задачей (TODO).

class Hobby(Task)
    Задачи-хобби.


ИНТЕРФЕЙС (interface.py)
------------------------
Глобальные переменные:
    TASK_TYPES      — список типов, берётся из task.TASK_TYPES
    TASKS_FILE      — путь к JSON-файлу хранилища ("tasks.json")
    HEADERS — словарь для шапки таблицы

Функции:
    task_to_dict(t)         -> dict     Преобразует задачу в словарь для json.
    task_from_dict(d)       -> Task     Конструирует задачу по json.
    load_tasks_from_file()  -> list     Читает tasks.json при старте; при ошибке — пустой список.
    save_tasks_to_file(tasks)           Перезаписывает tasks.json; вызывается после каждого добавления и при закрытии окна.

Экраны:
    Главный экран   — кнопки "Добавить задачу" и "Список задач".
    Форма ввода     — поля: дедлайн (DatePicker), время, минимальный интервал, тип (Dropdown).
                      is_done устанавливается в False автоматически.
    Таблица задач   — все поля Task.


ХРАНЕНИЕ ДАННЫХ
---------------
Задачи хранятся в tasks.json в следующем формате:
    [
      {
        "deadline":          "2026-05-01",
        "time_required":     3,
        "type":              "учёба",
        "min_cont_interval": 1,
        "is_done":           false
      },
      ...
    ]


ЦЕЛЕВЫЕ КОНСТАНТЫ (expectations.py)
------------------------------------
FreeTimePerWeek    — минимум свободного времени в неделю (часы)
AvgSatisfaction    — средняя удовлетворенность задаами
MaxLostImportance  — максимальная важность невыполненных задач
